# 장기 구현 계획: 서비스 확장성 전략

## 🎯 핵심 방향성

**개인 맞춤형 일정 관리 서비스에서 사용자 네트워크와 자연스럽게 연결되는 확장 가능한 서비스로 발전**

> 💡 **교수님 피드백 요약**: "상용 SNS" 기능은 단순히 SNS 서비스를 그대로 구현하라는 의미가 아니라, **서비스 확장성** 관점에서 제안된 의견입니다. 개인 맞춤형 기능을 유지하면서도 사용자 간 연결과 협업이 가능한 방향으로 확장하는 것이 목표입니다.

---

## 📌 확장성 관점에서 추가할 기능

### 1. 일정 공유 기능
- 친구/연락처 시스템 구축
- 일정 공유 권한 설정 (읽기 전용 / 수정 가능)
- 공유 일정 실시간 동기화
- 공유 링크 생성

**확장성 효과**: 개인 일정 관리 → 협업 도구로 확장

### 2. 일정 알림 메시지 시스템
- 일정 시작 전 알림 (15분 전, 1시간 전 등)
- 공유 일정 변경 알림
- 일정 참석 확인 요청
- 알림 메시지 타입별 분류

**확장성 효과**: 사용자 간 소통 강화 → 협업 효율성 향상

### 3. 외부 SNS 연동
- 일정을 외부 SNS에 공유 (Facebook, Twitter, Instagram 등)
- 외부 SNS 일정 가져오기
- 웹 공유 API 활용
- 공유 링크 생성 및 프리뷰

**확장성 효과**: 외부 플랫폼 사용자 유입 → 네트워크 확장

### 4. 캘린더 연동
- Google Calendar 연동 (OAuth 2.0)
- Outlook Calendar 연동
- ICS 파일 내보내기/가져오기
- 양방향 동기화 (shedAI ↔ 외부 캘린더)

**확장성 효과**: 기존 사용자 편의성 극대화 → 서비스 전환 장벽 낮춤

---

## 📊 단계적 확장 전략

1. **1단계**: 개인 일정 관리 (현재 상태) ✅
2. **2단계**: 일정 공유 + 알림 (사용자 간 연결 시작)
3. **3단계**: 외부 캘린더 연동 (기존 도구와 통합)
4. **4단계**: 외부 SNS 연동 (네트워크 확장)

**핵심 원칙**:
- 개인 맞춤형 기능은 유지
- 사용자가 원할 때만 네트워크 기능 사용
- 점진적 확장으로 사용자 부담 최소화

## 💡 결론

이 구현 계획은 shedAI를 **개인 맞춤형 일정 관리 서비스에서 사용자 네트워크와 자연스럽게 연결되는 확장 가능한 서비스**로 발전시키는 로드맵입니다.

**주요 가치**:
1. **개인 맞춤형 기능 유지**: 기존의 강점인 AI 맞춤형 일정 관리 기능은 계속 강화
2. **사용자 네트워크 형성**: 일정 공유, 알림 메시지 등을 통해 사용자 간 자연스러운 연결
3. **서비스 확장성**: 개인 일정 관리 → 협업 도구 → 외부 플랫폼 연동으로 점진적 확장
4. **사용자 편의성**: 기존 도구(캘린더, SNS)와의 연동으로 전환 장벽 최소화

이러한 방향으로 발전하면 **서비스 활용성과 확장성**이 크게 향상될 것입니다.
